Rules of my project

So even if i'm known to break all the rules while rnging... For my project and to create something uniques, i've decided to put some RUDE rules in order to... Except hating myself, i don't know. I could say it's to match the best my vision of Pok√©mon... I think.

  1. No PID Clones. It's the most important rule for me. If you're wondering what do i call a PID clone is that any Pokemon that shares the same PID with another one is considered as a clone. And so only one of them can be kept. Every result while doing RNG must stay UNIQUE.
  2. No eggs except if you CAN'T get it in the targeted gen. For example, a Bulbasaur in Gen 5 MUST be done by egg (and no transfer to have Gen 5 as original gen) but in Gen 7, it must be a wild, since it's available.
  3. Going for wilds rather than going for evolutions. If a Stage 1 or more Pokemon can be found in the wild, it'll be a priority compared to evolving the base stage Pokemon. The only exception is for my BASIC teams since i wanted to get them as soon as possible.
  4. Shiny Locks sucks. And since i want all the Shiny, i'm 300% fine with breaking them'all. No problem with that.
  5. Shiny Locked Pokemon are a priority compared to Events. For example, i'll keep a Shiny Zygarde done by RNG as stationary than keeping the Event one. Shiny Hunters old reflexs.
